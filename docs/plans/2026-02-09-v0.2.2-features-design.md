# BotDrop v0.2.2 Features Design

**Date:** 2026-02-09
**Status:** Design Complete, Ready for Implementation
**Estimated Effort:** 5-6 hours

## Overview

Three key features for v0.2.2 to improve user experience:

1. **Manual Update Check Button** - Allow users to check for updates anytime, especially useful if stuck in setup
2. **Dynamic Model Selector** - Searchable model picker that gets list from `openclaw models list`, reusable in Auth and Dashboard
3. **Config Template Cache** - Save user configuration (provider, model, API key, Telegram info) to avoid re-entering

## Background

### Current Pain Points

1. **No manual update trigger** - Users stuck in early setup steps can't access Dashboard to see update banner
2. **Hardcoded model selection** - Can't adapt to new models added to OpenClaw
3. **Repeated configuration** - Users must re-enter all info if OpenClaw config gets corrupted

### Design Goals

1. Update check accessible from any setup step
2. Model list dynamically fetched from OpenClaw, always up-to-date
3. Configuration cached in app for quick recovery
4. No breaking changes to existing setup flow

## Component Specifications

### 1. ConfigTemplateCache

**Purpose:** Cache user configuration to avoid re-entering information.

**File:** `app/src/main/java/app/botdrop/ConfigTemplateCache.java`

**Storage:** SharedPreferences (no encryption needed - convenience cache)

**Fields:**
```java
public class ConfigTemplate {
    public String provider;      // "google"
    public String model;         // "gemini-3-flash-preview"
    public String apiKey;        // "AIzaSy..."
    public String tgBotToken;    // "7123456:AAF..." (optional)
    public String tgUserId;      // "987654321" (optional)

    public boolean isValid() {
        return provider != null && !provider.isEmpty()
            && model != null && !model.isEmpty()
            && apiKey != null && !apiKey.isEmpty();
    }
}
```

**Methods:**
```java
public class ConfigTemplateCache {
    private static final String PREFS_NAME = "botdrop_config_template";

    // Save configuration template
    public static void saveTemplate(Context ctx, ConfigTemplate template);

    // Load configuration template
    public static ConfigTemplate loadTemplate(Context ctx);

    // Clear cache
    public static void clearTemplate(Context ctx);

    // Check if cache exists
    public static boolean hasTemplate(Context ctx);

    // Apply template to OpenClaw configuration files
    public static boolean applyTemplate(Context ctx, ConfigTemplate template) {
        BotDropConfig.setProvider(template.provider, template.model);
        BotDropConfig.setApiKey(template.provider, template.apiKey);
        if (template.tgBotToken != null) {
            BotDropConfig.setTelegramChannel(template.tgBotToken, template.tgUserId);
        }
        return true;
    }
}
```

**Error Handling:**
- Corrupted SharedPreferences â†’ Clear and return null
- Missing fields â†’ Validate with `isValid()`
- Version migration â†’ Support config version upgrade

### 2. ModelSelectorDialog

**Purpose:** Searchable model selector that fetches list from OpenClaw.

**File:** `app/src/main/java/app/botdrop/ModelSelectorDialog.java`

**UI Layout:**
```xml
<LinearLayout vertical>
    <TextView text="Select Model" />
    <EditText hint="Search models..." /> <!-- Real-time filter -->
    <RecyclerView /> <!-- Model list -->
    <TextView id="status" /> <!-- Loading/Error message -->
</LinearLayout>
```

**Workflow:**
1. Dialog shown â†’ Show loading state
2. Execute `openclaw models list` via BotDropService
3. Parse output into ModelInfo list
4. Display in RecyclerView
5. User types in search box â†’ Filter list in real-time
6. User clicks model â†’ Callback `onModelSelected(provider, model)` â†’ Close dialog

**Model Parsing:**

Input format:
```
Model                                      Input      Ctx      Local Auth  Tags
google/gemini-3-flash-preview              text+image 1024k    no    yes   default
anthropic/claude-sonnet-4-5                text+image 200k     no    yes
```

Parse to:
```java
public class ModelInfo {
    public String fullName;  // "google/gemini-3-flash-preview"
    public String provider;  // "google"
    public String model;     // "gemini-3-flash-preview"
    public String input;     // "text+image"
    public String context;   // "1024k"
    public boolean isDefault; // has "default" tag
}
```

**Search Logic:**
```java
private void filterModels(String query) {
    String lowerQuery = query.toLowerCase();
    List<ModelInfo> filtered = allModels.stream()
        .filter(m -> m.fullName.toLowerCase().contains(lowerQuery))
        .collect(Collectors.toList());
    adapter.updateList(filtered);
}
```

**Error Handling:**
- Command fails â†’ Show error message + retry button
- No models found â†’ Show "No models available"
- Parse error â†’ Show default model list fallback

**Callback Interface:**
```java
public interface ModelSelectedCallback {

```

## UI/UX Considerations

### Model Selector UX

1. **Loading State:** Show spinner while fetching models
2. **Empty State:** Clear message if no models found
3. **Error State:** Friendly error + retry button
4. **Search Feedback:** Real-time filtering as user types
5. **Selection Feedback:** Highlight selected model before closing

### Config Restore UX

1. **Non-intrusive:** Show as optional dialog, not forced
2. **Clear Action:** "Use Previous" vs "Start Fresh" buttons
3. **Skip if Broken:** If template invalid, don't show dialog
4. **Transparent:** Show what config will be restored

### Update Button UX

1. **Visual Feedback:** Disable during check, show loading
2. **Clear Result:** Toast or banner for success/failure
3. **Accessible:** Easy to spot but not intrusive
4. **Consistent:** Same style as other navigation buttons
